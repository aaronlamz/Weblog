> Time really does fly. Another year has passed, and it feels like the right moment to sum up a pretty ordinary one.

---

## 💻 Work

Twelve years into my career. Nothing dramatic—somewhere in the middle, as they say.

- **What I did this year**: Kept doing Web development as usual. I’m more open-minded about work now: less obsessed with grinding, more willing to try new things when they show up. Prefer a real work–life balance and not pushing myself too hard.
- **Technical growth**: Started using AI tools like **Cursor** and **Claude Code**, and got a taste of **Vibe Coding**. Writing code has shifted toward designing products with AI; efficiency has clearly gone up.
- **Vibe Coding**: AI is changing how we program. Using it made me see how powerful vibe coding can be. For straightforward tasks it can break down knowledge barriers and feel like a real “dimension drop.” That’s both risk and opportunity—only way through is to keep learning and grab the chance (classic “always hustling” mindset 😂).
- **Side gigs & side projects**: Tried some freelance and personal projects this year. Realized my time management still isn’t good enough; I need to plan better and get more done in less time.

---

## 🏠 Daily Life

### Routine

- **Sleep & health**: Having a 🐱 has helped—I’ve been going to bed and waking up earlier, so my schedule is more regular. As long as work from the night before doesn’t spill over 😅.

### Hobbies

- **Reading**: Still mostly technical books. I’d like to step away from work reading more and pick up other genres to broaden the view.
- **Fitness**: I’ve been casually working out for years with no big results—more for general wellness. Next year I might plan diet a bit more and aim for a more standard shape.

---

## ✈️ Travel

### Places I went

I traveled a bit more than in past years. Sticking to the “at least one trip a year” rule—life’s for experiencing.

1. **Altay road trip** — Scenery was great. Last day we got stuck in heavy snow on the road and sat in traffic till dawn. Unforgettable in its own way.
2. **Bangkok** — First time in Bangkok. Lots of eating and wandering; international vibe was nice.
3. **Tibet, Ngari (Ali) road trip** — As a Guangdong native, first time seeing thick snowfall. Every day was another “co” (lake), very photogenic. I had altitude sickness and threw up on day one, but the trip was still worth it.

### Takeaway

Getting out more, feeling different cultures and landscapes, does open your eyes.

---

## ❤️ Relationships

Not much to say here. I try to be kind and care more about family. With friends I’m working on better communication and expression. More socializing, more going out, meeting more people, trying more new things.

# ALL in AI: Why You Should Start Vibe Coding Right Now?

> The future of programming isn't writing more code, it's expressing your ideas better.

## Introduction: A Revolutionary Shift in Programming Paradigms

Remember when you first learned to code? Starting with Hello World, typing line by line, debugging syntax errors, reading documentation... This process often took months or even years to truly master.

But everything has changed.

In 2024, the maturity of AI programming tools reached a critical point. The emergence of Claude, GPT-4, Cursor, GitHub Copilot, and similar tools has transformed "programming in natural language" from science fiction into reality. This is **Vibe Coding** — an entirely new programming paradigm.

## What is Vibe Coding?

**Vibe Coding** is an AI-centric approach to programming, with the core philosophy:

> Shift from "writing code" to "conversing with AI," describing requirements in natural language and letting AI generate code.

### Traditional Programming vs Vibe Coding

| Traditional Programming | Vibe Coding |
|------------------------|-------------|
| Write every line manually | Describe needs in natural language |
| Search docs and Stack Overflow | Ask AI directly |
| Repeatedly debug syntax errors | AI generates correct code |
| Need deep understanding of details | Focus on architecture and logic |
| Steep learning curve | Quick start, learn by doing |

### Key Features

1. **Natural Language Interaction**: Describe what you want in human language, not how to implement it
2. **Rapid Prototyping**: From idea to prototype in minutes
3. **Lower Technical Barriers**: Even beginners can build complex applications
4. **Exponential Efficiency Gains**: Focus on creativity and architecture, not implementation details

We're at a pivotal moment:

- **Claude 3.5 Sonnet** and **GPT-4** code generation accuracy has reached professional levels

These tools have evolved from "experimental toys" into genuine productivity tools. If you haven't started yet, you're already behind.

**Developer B** (Vibe Coding):

A year later, who will have accumulated more project experience? Who will be more competitive?

**Early Adopter Advantages:**

- Individual developers can complete projects that previously required teams

### Scenario 3: Debugging and Optimization

**Traditional Debugging:**
```javascript
// Code breaks, you don't know why
function calculateTotal(items) {
  return items.reduce((sum, item) => sum + item.price, 0)
}
// TypeError: Cannot read property 'price' of undefined

// Start lengthy debugging...
// Breakpoints, console.log, docs...
```

**Vibe Coding Debugging:**
```
You: "This code throws an error, help me fix it"
AI: "The issue is that the items array may contain undefined or null values.
     Recommend adding filtering and type checking..."

You: "Give me a more robust version"
AI: [Generates improved code with error handling, type checking, edge cases]
```

You save not just time, but also energy and frustration.

4. Type safety: Missing TypeScript definitions

Suggested refactoring: [Detailed improved code]"

It's like having a senior engineer available 24/7 for code reviews.

## Common Misconceptions and Precautions

### Misconception 1: "AI Will Replace Programmers"

**Reality:**
- AI is a tool, not a replacement
- Just as calculators didn't replace mathematicians
- Cars didn't make us lose our legs
- AI frees us from repetitive work to focus on creation

**Humans Are Still Irreplaceable For:**
- **Creativity**: Product ideas and UX design
- **Judgment**: Tech selection and architecture decisions
- **Empathy**: Understanding user needs and pain points
- **Responsibility**: Code quality and security assurance

### Misconception 2: "Don't Need to Learn Programming Fundamentals"

**Why Fundamentals Still Matter:**

1. **Code Review**: You need to judge if AI-generated code is correct
2. **Debugging Skills**: Know how to locate issues
3. **Architecture Design**: Understand system design principles
4. **Communication**: Better express requirements to AI

**Recommended Learning Path:**
```
Learn by doing:
1. Build projects with AI (practice)
2. Understand AI-generated code (learn)
3. Consult docs for deeper concepts (consolidate)
4. Try implementing yourself (internalize)
```

### Misconception 3: "AI-Generated Code Is Always Correct"

**Common Issues:**

1. **Hallucinations**
   - AI may invent non-existent APIs
   - May use outdated syntax
   - Requires your verification and correction

2. **Security Vulnerabilities**
   - May lack input validation
   - May have SQL injection risks
   - Needs security review

3. **Performance Issues**
   - May use inefficient algorithms
   - May over-render
   - Needs performance testing

4. **Over-Engineering**
   - May add unnecessary abstractions
   - May over-optimize
   - Needs simplification

**Prevention Measures:**
```markdown
1. Always test AI-generated code
2. Use TypeScript for type safety
3. Write test cases
4. Code reviews
5. Security scanning
6. Performance monitoring
```
